## Overview
This room introduced the most common penetration testing frameworks and how each one fits different engagement types. Instead of focusing on exploitation, it explained how professional pentesters plan, structure, execute, and report assessments using standardized methodologies.

## Framework Selection

| Scenario | Framework |
|----------|-----------|
| Web Application | OWASP WSTG |
| Complete Pentest | PTES |
| Mobile Application | OWASP MASTG |
| Payment Card Environment | PCI DSS |

| U.S. Government | NIST SP 800-115 |
| Scientific / Measurable Results | OSSTMM |
| Cloud Environment | CSA CCM |

## Session Takeaways

- Different engagements require different frameworks.
- PTES provides a practical workflow for complete penetration testing.
- OWASP WSTG focuses specifically on web application security testing.
- MITRE ATT&CK complements penetration testing by mapping attacker behavior.
- Frameworks can be combined depending on the client's requirements.

## Lessons Learned

This room wasn't about learning new exploits or tools—it was about understanding how professional penetration tests are planned and structured. I learned that choosing the right framework depends on the target, regulations, and engagement objectives rather than using the same approach every time.
